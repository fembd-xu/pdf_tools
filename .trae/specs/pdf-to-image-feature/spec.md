# PDF 转图片功能 Spec

## 为什么 (Why)
用户需要一个简单易用的在线工具，将 PDF 文档转换为图片格式，以便于在不支持 PDF 查看的环境中分享或使用。特别是需要支持将多页 PDF 拼接为一张长图的功能，这在移动端阅读和分享长文档时非常实用。

## 变更内容 (What Changes)
- **项目初始化**: 使用 Vue 3 + TypeScript + Vite 构建全新的前端项目。
- **核心功能**: 集成 `pdfjs-dist` 库解析 PDF 文件。
- **导出模式**:
  1.  **逐页导出**: 将每一页 PDF 转换为独立的图片文件（如 PNG/JPEG）。
  2.  **长图导出**: 将所有 PDF 页面按顺序垂直拼接，生成一张完整的长图。
- **UI/UX**: 提供文件上传区域、转换进度条、预览区域和下载按钮。

## 影响 (Impact)
- **新功能**: 这是一个全新的项目，不影响现有代码。
- **依赖**: 需要引入 `pdfjs-dist` 用于 PDF 处理，`jszip` (可选) 用于批量下载图片，以及 UI 组件库 (如 Element Plus)。

## 新增需求 (ADDED Requirements)

### 需求: PDF 文件上传与解析
系统应允许用户上传本地 PDF 文件，并解析文件的页数和内容。

#### 场景: 上传成功
- **WHEN** 用户点击上传按钮并选择一个有效的 PDF 文件
- **THEN** 系统显示文件名、页数，并预加载第一页预览

### 需求: 导出为逐页图片
系统应支持将 PDF 的每一页单独转换为图片。

#### 场景: 逐页转换
- **WHEN** 用户选择“逐页导出”模式并点击转换
- **THEN** 系统生成对应页数的图片文件，并提供打包下载 (ZIP) 或逐张下载选项

### 需求: 导出为长图
系统应支持将所有 PDF 页面拼接为一张长图。

#### 场景: 长图转换
- **WHEN** 用户选择“长图导出”模式并点击转换
- **THEN** 系统将所有页面渲染并垂直拼接，生成一张包含所有内容的图片，并提供下载

## 界面设计 (UI Design)
- **顶部**: 简洁的导航栏/标题。
- **中间**: 
  - 大尺寸的文件上传区域（支持拖拽）。
  - 控制面板：选择导出模式（逐页/长图）、图片质量设置（可选）。
  - 预览区域：展示转换结果的缩略图。
- **底部**: 操作按钮（开始转换、下载结果）。
